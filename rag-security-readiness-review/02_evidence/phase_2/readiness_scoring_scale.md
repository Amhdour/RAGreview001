# Phase 2 Readiness Scoring Scale

## Readiness levels
- 0 = Not reviewed
- 1 = Identified only
- 2 = Partially reviewed
- 3 = Reviewed with gaps
- 4 = Evidence-supported
- 5 = Evidence-supported and tested

## Readiness categories
- Authentication
- Authorization
- Retrieval ACL
- Document ingestion
- Prompt-injection resilience
- Citation/source integrity
- Connector security
- Agent/tool governance
- MCP security
- Sandbox/file safety
- Secrets handling
- Logging safety
- Telemetry safety
- CI/CD security
- Supply-chain security
- Deployment security
- Test coverage
- Evidence quality
- Residual-risk clarity

## Scoring guidance
- Use 0 when no meaningful review has occurred.
- Use 1 when the area has been identified but not assessed.
- Use 2 when some source evidence exists but major gaps remain.
- Use 3 when the area has been reviewed but relevant gaps or unknowns remain.
- Use 4 when evidence supports the assessment from source review.
- Use 5 only when evidence-supported review is paired with direct testing evidence.

## Interpretation rule
A higher readiness score does not imply absence of risk. It only reflects the depth and quality of evidence available for the category.
