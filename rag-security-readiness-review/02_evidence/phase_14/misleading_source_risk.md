
# Misleading Source Risk

| Misleading source path | File/path evidence | What is confirmed | What is unverified | Potential impact | Later validation method |
| ---------------------- | ------------------ | ----------------- | ------------------ | ---------------- | ----------------------- |
| Title mismatch | `backend/onyx/connectors/models.py`; `backend/onyx/connectors/web/connector.py`; `web/src/components/search/results/Citation.tsx` | Connector code can set titles independently of URLs and semantic identifiers | Whether title/identifier mismatches are surfaced or detected | Users may trust the wrong source name | Validate title/identifier consistency in end-to-end tests |
| URL mismatch | `backend/onyx/connectors/web/connector.py`; `backend/onyx/context/search/models.py` | URLs are copied from source links into search docs | Whether displayed links always match canonical source records | Broken or deceptive document links | Compare rendered links against canonical document records |
| Connector source mismatch | `backend/onyx/db/models.py`; `backend/onyx/db/document.py`; `backend/onyx/context/search/models.py` | Source and document data move through multiple layers and can be updated independently | Whether mismatched connector state is caught before display | A source may be shown as the wrong connector type | Test connector updates and display reconciliation |
| Chunk/source mismatch | `backend/onyx/context/search/models.py`; `backend/onyx/tools/tool_implementations/utils.py` | Chunk-to-document conversion keeps document ID and title together | Whether a chunk from one document could be rendered as another source under stale state | Misattributed evidence | Validate chunk/document reconciliation after updates |
| Answer loosely associated with source | `backend/onyx/chat/citation_processor.py`; `backend/onyx/chat/citation_utils.py` | The backend maps citation numbers to documents but does not prove semantic answer fidelity | Whether the answer text actually matches the cited source content | Overstated trust in cited evidence | Cross-check answer claims with source text in runtime tests |
| Ranking/confidence ambiguity | `backend/onyx/context/search/models.py`; `backend/onyx/server/query_and_chat/session_loading.py` | Search results are sorted and displayed, but confidence semantics are not a display guarantee | Whether users can distinguish ranking from authoritative support | Weak evidence may be overread as strong support | Validate source-confidence presentation and labels |
